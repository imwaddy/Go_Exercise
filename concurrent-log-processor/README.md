# Concurrent Log Processor

## Problem Statement

You are given application logs generated by multiple services. Each log entry contains:

```text
<timestamp> <service_name> <response_time_ms>
```

Your task is to compute the following metrics for each service:

1. Total number of requests
2. Average response time
3. Maximum response time
4. 95th percentile response time (P95)

Finally, output all services sorted in descending order of P95 latency.

---

## Input Format

The first line contains an integer:

```text
N
```

representing the number of log entries.

The next `N` lines contain log records in the format:

```text
timestamp service_name response_time_ms
```

### Example

```text
12
100 auth 120
101 payment 300
102 auth 150
103 auth 200
104 payment 250
105 inventory 80
106 auth 110
107 payment 400
108 inventory 90
109 inventory 100
110 payment 350
111 auth 170
```

---

## Output Format

For each service print:

```text
service_name total_requests average_response_time maximum_response_time p95_latency
```

Services must be sorted by:

1. Highest P95 latency first
2. If P95 latency is the same, sort by service name in ascending order

Average response time must be printed with exactly 2 decimal places.

---

## P95 Calculation

For a service having `n` response times:

1. Sort the response times in ascending order.
2. Compute:

```text
index = ceil(0.95 * n) - 1
```

3. P95 latency is the value at that index.

### Example

Response times:

```text
[110, 120, 150, 170, 200]
```

Calculation:

```text
n = 5

index = ceil(0.95 * 5) - 1
      = ceil(4.75) - 1
      = 5 - 1
      = 4
```

P95 latency:

```text
200
```

```text

```

---

## Constraints

```text
1 <= N <= 1,000,000
1 <= response_time_ms <= 100,000
timestamp >= 0
service_name contains no spaces
```

---

## Follow-Up

Implement the solution using concurrent processing:

* Read logs from input.
* Use a worker pool with `M` goroutines.
* Distribute log entries through a channel.
* Aggregate metrics safely.
* Produce the same final output.

The follow-up should focus on:

* Goroutines
* Channels
* Synchronization
* Memory efficiency
* Concurrent aggregation design

---

## Expected Complexity

A good solution should achieve approximately:

```text
Time Complexity: O(N log N)
Space Complexity: O(N)
```

where `N` is the total number of log entries.
